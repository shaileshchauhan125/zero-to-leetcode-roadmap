üß© Problem 1: Container With Most Water

You are given an array of heights representing vertical lines. Find two lines that together with the x-axis form a container that holds the most water.

Input: [1,8,6,2,5,4,8,3,7]
Output: 49 (lines at indices 1 and 8)
‚úÖ Correct! Two Pointers is the optimal approach

Key insight: We need to find the maximum area between two lines. Starting with pointers at both ends and moving the shorter line inward ensures we don't miss the optimal solution.

üéØ Solution Approach:
  1. Place one pointer at the beginning, one at the end
  2. Calculate area: min(height[left], height[right]) √ó (right - left)
  3. Move the pointer with the smaller height inward
  4. Track the maximum area found
Time Complexity: O(n) - Single pass through array
Space Complexity: O(1) - Only using two pointers
üß© Problem 2: Longest Substring Without Repeating Characters

Given a string, find the length of the longest substring without repeating characters.

Input: "abcabcbb"
Output: 3 (substring "abc")
‚úÖ Correct! Sliding Window is perfect here

Key insight: We need to find the longest valid substring, which requires expanding and contracting a window based on the duplicate constraint.

üéØ Solution Approach:
  1. Use a hashmap to track character positions
  2. Expand window by moving right pointer
  3. When duplicate found, shrink window from left
  4. Track maximum window size encountered
Time Complexity: O(n) - Each character visited at most twice
Space Complexity: O(min(m,n)) - Size of character set
üß© Problem 3: Search in Rotated Sorted Array

You are given a sorted array that has been rotated at some pivot. Search for a target value in O(log n) time.

Input: nums = [4,5,6,7,0,1,2], target = 0
Output: 4 (index of target)
‚úÖ Correct! Binary Search handles rotation elegantly

Key insight: Even though the array is rotated, we can still eliminate half the search space in each iteration by identifying which half is properly sorted.

üéØ Solution Approach:
  1. Compare middle element with left and right bounds
  2. Determine which half is properly sorted
  3. Check if target lies in the sorted half
  4. Eliminate the half that doesn't contain target
Time Complexity: O(log n) - Binary search behavior maintained
Space Complexity: O(1) - No extra space needed
