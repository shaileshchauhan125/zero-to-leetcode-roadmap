üî¨ Optimization Fundamentals

Optimization is both an art and a science. It's about making informed decisions on where to invest your effort for maximum performance gains. Let's start with the core principles that guide all optimization decisions.

üìä

Measure First, Optimize Second

Never optimize without data. Profile your code to identify real bottlenecks, not perceived ones.

Example: 80% of execution time often comes from 20% of the code
‚öñÔ∏è

Understand Tradeoffs

Every optimization has costs. Learn to balance time, space, readability, and maintainability.

Example: Hash tables trade space for time complexity improvements
üéØ

Context Matters

The best optimization depends on your constraints: data size, memory limits, and use patterns.

Example: Different algorithms excel with small vs. large datasets
‚ö†Ô∏è

Avoid Premature Optimization

Donald Knuth's famous quote: "Premature optimization is the root of all evil." Optimize when you have proof it's needed.

Example: Write clean, correct code first, then optimize hot paths

‚öñÔ∏è Space-Time Tradeoffs Mastery

The heart of optimization lies in understanding space-time tradeoffs. Sometimes using more memory dramatically improves speed, and sometimes the opposite is true. Master these decisions and you'll think like an expert optimizer.

üíæ Memoization Example

Before (Exponential Time)
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

# Time: O(2^n), Space: O(n)
After (Linear Time)
def fibonacci(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fibonacci(n-1, memo) + fibonacci(n-2, memo)
    return memo[n]

# Time: O(n), Space: O(n)
Key Insight: Trading O(n) space for exponential time improvement - one of the most powerful optimizations in computer science!

üèóÔ∏è Preprocessing Example

Raw Queries
def range_sum(arr, left, right):
    return sum(arr[left:right+1])

# Setup: O(1), Query: O(n)
Prefix Sum Array
class PrefixSum:
    def __init__(self, arr):
        self.prefix = [0]
        for num in arr:
            self.prefix.append(self.prefix[-1] + num)
    
    def range_sum(self, left, right):
        return self.prefix[right+1] - self.prefix[left]

# Setup: O(n), Query: O(1)
Key Insight: Upfront O(n) preprocessing cost pays dividends when you have multiple queries!

üèÜ Master Challenge: Maximum Subarray

Problem: Find the maximum sum of a contiguous subarray.

Example: [-2, 1, -3, 4, -1, 2, 1, -5, 4] ‚Üí Maximum sum: 6

Step 1: Brute Force O(n¬≥)
def max_subarray_brute(nums):
    max_sum = float('-inf')
    n = len(nums)
    
    for i in range(n):
        for j in range(i, n):
            current_sum = 0
            for k in range(i, j + 1):
                current_sum += nums[k]
            max_sum = max(max_sum, current_sum)
    
    return max_sum
Time: O(n¬≥), Space: O(1)
Step 2: Optimized O(n¬≤)
def max_subarray_optimized(nums):
    max_sum = float('-inf')
    n = len(nums)
    
    for i in range(n):
        current_sum = 0
        for j in range(i, n):
            current_sum += nums[j]
            max_sum = max(max_sum, current_sum)
    
    return max_sum
Time: O(n¬≤), Space: O(1)
Step 3: Kadane's Algorithm O(n)
def max_subarray_kadane(nums):
    max_sum = current_sum = nums[0]
    
    for num in nums[1:]:
        current_sum = max(num, current_sum + num)
        max_sum = max(max_sum, current_sum)
    
    return max_sum
Time: O(n), Space: O(1)
